An \code{EventNewThread} triggers when a process spawns a new thread. The Event
class\' getThread function returns the original Thread that performed the spawn
operation, while \code{EventNewThread}\'s getNewThread returns the newly created
Thread.

This event is never instantiated by ProcControlAPI and simply serves as a
place-holder type for a user to deal with thread creation without having to deal
with the specifics of LWP and user thread creation.

A callback function that receives an \code{EventNewThread} can use the two field
form of \code{Process::cb\_ret\_t}  to control the parent and child thread.

Associated EventType Codes: \code{ThreadCreate}, \code{UserThreadCreate}, and
\code{LWPCreate}

\begin{apient}
Thread::const_ptr getNewThread() const
\end{apient}

\apidesc{
This function returns a const pointer to the Thread object that represents the
newly spawned thread.
}
